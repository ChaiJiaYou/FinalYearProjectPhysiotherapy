## 🔄 文档维护

### 如何更新文档

1. **功能新增时**
   - 更新主 README.md 的功能列表
   - 在 FEATURES/ 中创建新功能文档
   - 更新 API_DOCUMENTATION.md 添加新接口

2. **优化改进时**
   - 更新 OPTIMIZATION_SUMMARY.md
   - 更新相关技术文档的性能数据

3. **架构变更时**
   - 更新 ARCHITECTURE.md
   - 更新架构图和流程图

### 文档版本控制

建议使用 Git 标签标记文档版本：
```bash
git tag -a docs-v1.0 -m "Initial documentation release"
git push origin docs-v1.0
```
