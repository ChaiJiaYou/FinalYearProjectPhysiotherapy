### 常见评审问题准备

**Q1: 为什么选择 DTW 而不是深度学习？**
```
回答要点（参考 ARCHITECTURE.md）:
✅ 只需 3-5 个样本（深度学习需要数百个）
✅ 训练时间 < 2 秒（深度学习需要小时级）
✅ 可解释性强（距离度量 vs 黑盒）
✅ 适合个性化康复训练场景
```

**Q2: 系统的准确率如何验证？**
```
回答要点（参考 OPTIMIZATION_SUMMARY.md）:
✅ 单元测试: 17 个测试用例
✅ 端到端测试: 完整工作流验证
✅ 实际测试: 稳定性从 60% 提升到 92%
✅ 优化数据: IN-OUT 抖动 ↓80%, 重复计数 ↓70%
```

**Q3: 如何处理不同用户的个体差异？**
```
回答要点（参考 ACTION_LEARNING_TECHNICAL.md）:
✅ 自适应归一化: 消除身高、体型差异
✅ 粘性追踪: 适应不同摄像头角度
✅ 多窗口 DTW: 适应不同执行速度
✅ 个性化模板: 每个用户可以创建自己的动作
```

**Q4: 系统的实时性如何？**
```
回答要点（参考 ACTION_LEARNING_TECHNICAL.md）:
✅ 识别延迟: < 100ms/帧
✅ 识别帧率: 10 FPS
✅ DTW 优化: Sakoe-Chiba 带约束（6.7× 加速）
✅ 内存占用: < 70 KB/动作
```

**Q5: 未来如何扩展？**
```
回答要点（参考 README.md + ARCHITECTURE.md）:
✅ 深度学习模式（已预留接口）
✅ 多人同时识别
✅ 动作质量评估
✅ 移动端应用
✅ 云端部署（可扩展架构）
```
